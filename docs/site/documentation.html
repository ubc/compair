Documentation

ComPAIR is an open-source learning application being developed by the University of British Columbia's Centre for Teaching, Learning & Technology (CTLT) using a GPL-3.0 license. ComPAIR is an official university tool available for all internal faculty and staff to use (with full support) in live courses. Other educational institutions are welcome to download, deploy, and add to the code.

ComPAIR is available internally at UBC via Connect and externally to other institutions via GitHub

UBC faculty and staff

Faculty and staff at UBC can get started with ComPAIR by using UBC's learning management system, Connect. Through Connect, you can set up a course in ComPAIR that will automatically be populated with your class list.

  1. Set up an account either by adding ComPAIR as an LTI tool in Connect or contacting LT Hub for assistance with this process.
  2. Once you have an account, click on the LTI link in Connect or log in with your CWL at: compair.elearning.ubc.ca.

Outside UBC

Outside of UBC, you will need to set up your own instance of the application. A ComPAIR instance requires a minimum of a web server running Python 2.7 or 3 and a MySQL or MariaDB database. We also recommend running Redis and a Celery daemon to reduce server load for certain tasks.

  • CentOS 7 Server (Standalone Linux): Set up an all-in-one ComPAIR server. Useful for piloting ComPAIR or small scale production environments.
  • Amazon Web Services (AWS): Set up ComPAIR on Amazon Web Services. Useful for larger scale production environments.
  • Google Container Engine / Kubernetes: Set up ComPAIR on Google Container Engine or Kubernetes. Useful for larger scale production environments.
  • Development Instance via Docker: Set up a development environment for ComPAIR. Useful for contributing to ComPAIR or customizing code. Not recommended for use in a production environment. (Instead, follow the Kubernetes guide above.)
